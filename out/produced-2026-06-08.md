# df-kpm-Portfolio-Tracker

## Übersicht

**df-kpm-Portfolio-Tracker** ist ein Python-basiertes Portfolioüberwachungs-Tool, das täglich ein End-of-Day NAV (Net Asset Value) Snapshot für alle Anlageklassen erstellt und Abweichungen in der Allokation gegen die Variante-D Zielgewichte detektiert. Das Tool implementiert Kelly-Fraktions-Sizing (0.25–0.40 adaptive), eine dreistufige Drawdown-Begrenzung (soft 15 % / hard 20 % / no-go 25 %) und HMAC-SHA256-Audit-Logging für jede Aktion. Standardmäßig läuft es in einem Sandbox-Modus mit einem simulierten Dreiteiler-Portfolio (Aktien / Anleihen / Bargeld), das erst realen Marktdaten zugreift, wenn eine explizit geprüfte Phronesis-Kennung aktiviert wird.

## Installation

### Voraussetzungen

- Python 3.9+
- macOS (für LaunchAgent-Scheduling) oder jedes Unix-basierte Betriebssystem

### Schritte zur Installation

1. **Repository klonen**

   ```bash
   git clone https://github.com/meokemmer-jpg/df-kpm-portfolio-tracker.git
   cd df-kpm-portfolio-tracker
   ```

2. **Erstellen und aktivieren eines virtuellen Umgebung**

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. **Installieren von Abhängigkeiten**

   ```bash
   pip install pytest
   ```

4. **Konfigurieren der Umgebungsvariablen** (Sandbox-Modus erfordert keine zusätzliche Konfiguration)

   ```bash
   # Standard — sandbox-Mock-Portfolio, kein echter Markt-Datenzugriff
   export DF_KPM_PORTFOLIO_REAL_ENABLED=false

   # Real-Modus — explizite Phronesis-Kennung erforderlich für den Zugriff auf echte Marktdaten
   export DF_KPM_PORTFOLIO_REAL_ENABLED=true
   export PHRONESIS_TICKET="PT-YYYY-MM-DD-001"
   export OPERATOR_SIGNOFF_ID="your-signoff-id"
   ```

5. **Starten des Trackers** (SQLite-Zustand-Datenbank wird automatisch erstellt, wenn es das erste Mal ausgeführt wird)

   ```bash
   python3 -m src.adapter_orchestrator
   ```

6. **(Optional) Einrichtung des macOS LaunchAgents für tägliche 06:00-Uhr-Verwaltung**

   ```bash
   cp scripts/com.kemmer.df-kpm-portfolio-tracker.plist ~/Library/LaunchAgents/
   launchctl load ~/Library/LaunchAgents/com.kemmer.df-kpm-portfolio-tracker.plist
   ```

## Benutzung

### 1 — Sandbox NAV-Aufnahme (Standard)

Keine zusätzliche Umgebungs-Konfiguration nötig. Führen Sie den Befehl aus:

```bash
python3 -m src.adapter_orchestrator
```
Dies startet das Programm im Sandbox-Modus, ohne echten Markt-Datenzugriff.

### 2 — Real NAV-Aufnahme (mit Phronesis-Berechtigung)

Für die Verwendung des realen Marktdaten-Zugriffs muss eine spezifische Umgebungsvariable gesetzt sein:

```bash
export DF_KPM_PORTFOLIO_REAL_ENABLED=true
```

Es müssen auch zusätzliche Berechtigungen wie `PHRONESIS_TICKET` und `OPERATOR_SIGNOFF_ID` konfiguriert werden, um die realen Datenbankzugriffe zu aktivieren. Diese Schritte sind notwendig, um die Sicherheitsrichtlinien der Familie Kemmer zu gewährleisten.
